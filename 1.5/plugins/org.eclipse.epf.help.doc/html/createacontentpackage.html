Create a Content Package

Method content is organized into content packages which, in turn, are contained in method plug-ins.

Before creating a content package, you should create a method plug-in. For more information, see Create a Method Plug-in.
You should always create a new Method Content Package and Method Content in a Method Plug-in that you produce. This separates your content from original content shipped with the tool and allows you to update your own library with new library releases without affecting the content that you have created in your own plug-ins.
Note: You cannot create a new content package or any other element in a locked plug-in.

To create a content package:

  1. Find your method plug-in in the Library View.
  2. Drill into the plug-in to locate the package called Content Packages. This package contains all packages that are allowed to contain method elements.
  3. Right click the content package and then select New > Content Package. The content package editor appears.
  4. Under General Information, give the new content package a unique name and provide a brief description.
  5. Save the new content package in one of three ways:
    1. Click the disk icon in the toolbar.
    2. Select File > Save.
    3. Simply close the content editor.
Note: The tool automatically creates nodes for Task, Role, Work Product, and Guidance under the new content package.
