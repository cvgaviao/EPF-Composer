Create a Method View with Custom Categories

Custom categories allow you to categorize content according to any scheme you want. Custom categories can then be used to compose publishable views, providing a means to organize method content prior to publishing.

To create a new custom category:

  1. Switch to the Authoring Perspective.
  2. Open the Method Content folder in the method plug-in where you want to create the custom category.
  3. Right click the Custom Categories folder and select New > Custom Category. The editor window opens.
  4. Provide descriptive information about the custom category in the fields available under the Description tab.
  5. Under the Assign tab, you can select the things you want to include in the custom category. Select individual method elements, processes, standard categories or custom categories. The custom category is also available to add to individual elements on the Categories tab in the appropriate editor.
  6. Once you assign content to the custom category, you can modify the sequence of the items in the category. This affects the sequence of the items in the Library View and Configuration View, as well as in a published Web site if the custom category is included as a view in a configuration.
  7. To save your new custom category, click File > Save or close the editor and click Yes when prompted to save the changes.
You can now use the custom category as a view in a published configuration.
  1. Switch to the Authoring Perspective.
  2. In the Library View, double click the configuration to which you want to add a view. The configuration editor opens.
  3. Click the Views tab and then click the Add View button. The Select Categories dialog appears. The Select Categories dialog displays all the standard and custom categories in the configuration.
  4. Select one or more categories to include as a view in your configuration and then click OK. The view display provides a preview of how the view will appear n the published tree browser.
  5. Select a view to display as the default start-up view in the published Web site, and then click the Make Default button.
  6. To save the configuration, click File > Save or close the configuration editor and click Yes when prompted to save the changes.
