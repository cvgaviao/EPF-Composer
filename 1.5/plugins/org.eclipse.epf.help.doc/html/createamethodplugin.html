Create a Method Plug-in

Method Plug-ins allow the extension or customization of base library content without directly changing that base content by defining the differences to the content. You should always create new content in a method plug-in that you produce to separate your content from the original OpenUP/Basic content. This also allows you to update your own library with new releases of the OpenUP/Basic Method Library without affecting the content that you have created in your own plug-ins.

Note: Plug-ins that are shipped with Eclipse Process Framework Composer are locked and read-only. These plug-ins are grayed out in the Library View, indicating that they are locked.
To create a new Method Plug-in:
  1. Make sure you are in the Authoring Perspective.
  2. You can begin the method plug-in creation process in one of two ways:
    • Click File > New > Method Plug-in. Or,
    • In the Library view, right click an existing plug-in and select New Method Plug-in.
    The New Method Plug-in wizard opens.
  3. Type a name for your new plug-in in the Name field and then select a plugin in the Referenced Plug-ins field. The plug-in you create extends the plug-in you select from the list and allows you to use content from the selected plug-in as you create your own plug-in content.
  4. Provide a brief description of the plug-in and list its authors to help you and others keep track of the plug-ins in the library. You can return to edit this information at any time.
  5. Click Finish. Your new plug-in now appears in the Library View with the other Method Plug-ins.
