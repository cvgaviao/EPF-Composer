Exercise 2: Explore a Process using Authoring perspective

The purpose of this tutorial is to explore the information and editors for processes. A Capability Pattern is used as an example but the same applies to Delivery Processes.

  1. Switch to Authoring perspective.

  2. In the Library View, click RUP > Processes > Capability Patterns > Classic RUP > Discipline Workflows.

  3. Double click the Analysis and Design Capability Pattern and examine the Capability Pattern.

  4. The Description tab provides a number of sections that can be used to fully describe the Capability Pattern.

    A Capability Pattern or Delivery Process must be associated with at least one configuration but can be associated with many. The configuration sets the scope for core method content and other process elements that can be included in a process.

  5. Click the Work Breakdown Structure tab.

    This shows the Activities in the Capability Pattern and the (core method content) tasks that are included in each Activity.

    When a task is added to a process, the roles and work products that are associated with the task are also pulled into the process.

    Each task, role, and work product shown in the activity is actually an instance of the element as it is used at the particular point in the process. So what you see in the process are actually Task Descriptors, Role Descriptors and Work Product Descriptors.

  6. Click the Team Allocation tab.

    This shows the activities in the capability pattern and the roles that are the performers of the tasks in the activities.

  7. Click the Work Product Usage tab.

    This shows the activities in the capability pattern and the work products that are input and outputs of the tasks in the activities.

  8. Click on the Consolidated View tab.

    This shows the full set of information for each activity in the Capability Pattern. It shows the tasks, work products and roles within each activity.

  9. Right click any task in the Work Breakdown Structure view, and click on Show Properties View.

    This displays a new view that shows all the information about the task (task descriptor) as it is used in the process and allows you to customize the task as it is used at this point in the process.

    The Properties View has a number of tabs that organize the task descriptor information.

  10. Click on the Documentation tab. This allows you to enter information that is specific to this Task Descriptor.

  11. Click on the Roles tab. This allows you to replace, remove or add to the roles that are associated with the core method task for this specific instance of the task in the process.

  12. Click on the Work Products tab. This allows you to replace, remove or add to the work products that are associated with the core method task for this specific instance of the task in the process.

  13. Click on the Steps tab. This allows you to adjust the task steps as they are applied at this point in the process.

  14. When a role is selected in the Team Allocation view, or a work product is selected in the Work Product Usage view, the Properties View allows you to edit these elements in the context of the process.

  15. Click on an Activity in the process.
    The properties view allows you to see roll-ups of the tasks, roles, and work products in the activity. Roll-ups can also be displayed in the main content window by selecting the Roll-Up option in the context sensitive menu. The Roll-Up view can be turned off by selecting, Roll-Down.
