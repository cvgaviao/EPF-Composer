Exercise 2: Create Work Products

The purpose of this exercise is to create new method content. Specifically, Work Products of type Artifact . You can create any content but some examples are provided if you want to use them. These examples are referred to in other tutorials.

Method and Process elements have two names in EPF Composer: Name and Presentation Name.

The Name field is used as the file name for the item. In RUP, we are careful how we name the files and use conventions such as lower case, no spaces, and no special characters. We do this so that the name is a valid file name on all operating systems and for integration with source control software in the future. The Name is shown in the Library View.

The Presentation Name is the name that is shown on published pages and it is also shown in the Configuration View. This name can contain any characters, but by convention in RUP, we use the same text as the Name, just prettier.

In the Library view you can choose to display Names or Presentation Names. You can switch the name display using the ‘Show Presentation Names' button in the Library view toolbar.

In general, you create new elements by right-clicking the folder that will contain the new element. For example, to create a new role in a content package, right-click the Roles folder, highlight New and then click Role.

  1. Make sure you are in the Authoring perspective.

  2. Within My Content Package right-click on the Work Products folder.

  3. Highlight New and then click Artifact. The artifact editor is displayed.

  4. In the Name field, type the new Artifact name. In this instance, we'll call it my_rup_navmap.

  5. In the Presentation Name field, type:

    6. My RUP Navigation Map

  6. In the Brief Description field, type:

    The Navigation Map expresses the structure of the user-interface elements in the system, along with their potential navigation pathways.

  7. In the Purpose field type:

    There is one Navigation Map per system. The purpose of the Navigation Map is to express the principal user interface paths through the system. These are the main pathways through the screens of the system and not necessarily all of the possible paths. It can be thought of as a road map of the system's user interface. The Navigation Map makes it easy to see how many "clicks" it will take a user to get to a specific screen.

  8. Click the preview tab to view the new Artifact.

  9. Save the Artifact.

  10. Repeat steps 2 through 9 using the following information:

    •  Artifact Name: my_rup_user_interface_prototype

    •  Presentation Name: My RUP UI Prototype

    •  Brief Description: A user-interface prototype is an example of the user interface that is built in order to explore and/or validate the user-interface design.

    •  Purpose: The following roles use the user-interface prototype:

    - user-interface designers , to explore and/or validate the user-interface design before too much is invested in it

    - requirements specifiers , to understand the user interface for a Use Case

    - system analysts , to understand how the user interface impacts the analysis of the system

    - designers , to understand how the user interface impacts and what it requires from the "inside" of the system

    - managers , to plan development and testing activities
